# SmartSync 接口文档

## 1. 文档说明

- 项目名称：`SmartSync`
- 接口风格：RESTful JSON
- 默认服务地址：`http://localhost:8080`
- Swagger 地址：`http://localhost:8080/swagger-ui.html`
- OpenAPI 地址：`http://localhost:8080/v3/api-docs`

除登录注册接口外，其余接口默认需要携带 JWT。

请求头格式：

```http
Authorization: Bearer <token>
Content-Type: application/json
```

## 2. 统一返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "OK",
  "data": {}
}
```

字段说明：

- `code`：业务状态码
- `message`：业务提示信息
- `data`：业务数据，失败时通常为空

## 3. 通用状态码

### 3.1 基础状态码

| code | message | 说明 |
| --- | --- | --- |
| 200 | OK | 请求成功 |
| 400 | 请求参数错误 | 参数缺失、格式非法 |
| 401 | 未认证或凭证失效 | 未带 token 或 token 无效 |
| 403 | 无权限访问 | 已登录但权限不足 |
| 404 | 资源不存在 | 查询不到目标记录 |
| 409 | 资源冲突 | 通用冲突码 |
| 500 | 服务器内部错误 | 未预期异常 |

### 3.2 业务状态码

| code | message | 说明 |
| --- | --- | --- |
| 4101 | 终端未启用或待审核 | 终端状态不是启用 |
| 4102 | 终端编码已存在 | 注册时终端编码重复 |
| 4103 | 终端编码或密钥错误 | 登录失败 |
| 4201 | RFID 格式非法 | 长度或字符集非法 |
| 4202 | RFID 校验位不匹配 | RFID 校验失败 |
| 4203 | RFID 未绑定任何患者 | 校验通过但未查到患者 |
| 4301 | 该身份证号已存在患者档案 | 患者身份证号冲突 |
| 4302 | 该 RFID 已绑定其他患者 | RFID 已被其他患者占用 |

## 4. 分页返回结构

分页接口的 `data` 结构如下：

```json
{
  "total": 1,
  "current": 1,
  "size": 10,
  "records": []
}
```

字段说明：

- `total`：总记录数
- `current`：当前页码，从 1 开始
- `size`：每页条数
- `records`：当前页数据列表

## 5. 终端鉴权接口

基础路径：`/api/auth`

### 5.1 终端注册

- 方法：`POST`
- 路径：`/api/auth/register`
- 鉴权：否

请求体：

```json
{
  "terminalCode": "T001",
  "terminalName": "住院部3楼-RFID读卡器",
  "secretKey": "secret123"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| terminalCode | string | 是 | 终端编码，2 到 64 位，唯一 |
| terminalName | string | 否 | 终端显示名称，最大 128 位 |
| secretKey | string | 是 | 终端密钥，8 到 128 位 |

成功响应：

```json
{
  "code": 200,
  "message": "OK",
  "data": 1926308149684289538
}
```

说明：

- `data` 为新建终端主键
- 注册后默认状态为 `0`，需要管理员审核后才能登录

可能返回：

- `4102`：终端编码已存在
- `400`：参数校验失败

### 5.2 终端登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 鉴权：否

请求体：

```json
{
  "terminalCode": "T001",
  "secretKey": "secret123"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| terminalCode | string | 是 | 终端编码 |
| secretKey | string | 是 | 终端密钥 |

成功响应：

```json
{
  "code": 200,
  "message": "OK",
  "data": {
    "token": "<JWT_TOKEN>",
    "expiresIn": 7200,
    "terminalCode": "T001"
  }
}
```

响应字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| token | string | JWT token |
| expiresIn | number | 有效期，单位秒 |
| terminalCode | string | 当前登录终端编码 |

可能返回：

- `4101`：终端未启用或待审核
- `4103`：终端编码或密钥错误
- `400`：参数校验失败

## 6. 终端管理接口

基础路径：`/api/terminal`

### 6.1 分页查询终端

- 方法：`GET`
- 路径：`/api/terminal/page`
- 鉴权：是

查询参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| current | number | 否 | 1 | 当前页 |
| size | number | 否 | 10 | 每页条数 |
| keyword | string | 否 | - | 模糊匹配终端编码或名称 |

示例请求：

```http
GET /api/terminal/page?current=1&size=10&keyword=T00
Authorization: Bearer <token>
```

成功响应：

```json
{
  "code": 200,
  "message": "OK",
  "data": {
    "total": 1,
    "current": 1,
    "size": 10,
    "records": [
      {
        "id": 1926308149684289538,
        "terminalCode": "T001",
        "terminalName": "住院部3楼-RFID读卡器",
        "secretHash": "$2a$10$xxxxx",
        "status": 1,
        "lastLoginAt": "2026-05-02T13:00:00",
        "createdAt": "2026-05-02T12:30:00",
        "updatedAt": "2026-05-02T13:00:00",
        "deleted": 0
      }
    ]
  }
}
```

### 6.2 查询终端详情

- 方法：`GET`
- 路径：`/api/terminal/{id}`
- 鉴权：是

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 终端主键 |

成功响应：

```json
{
  "code": 200,
  "message": "OK",
  "data": {
    "id": 1926308149684289538,
```

### 6.3 修改终端状态

- 方法：`PUT`
- 路径：`/api/terminal/{id}/status`
- 鉴权：是

查询参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| status | number | 是 | `0` 待审核，`1` 启用，`2` 禁用 |

示例请求：

```http
PUT /api/terminal/1926308149684289538/status?status=1
Authorization: Bearer <token>
```

成功响应：

```json
{
  "code": 200,
  "message": "OK"
}
```

可能返回：

- `400`：非法状态值
- `404`：资源不存在

### 6.4 删除终端

- 方法：`DELETE`
- 路径：`/api/terminal/{id}`
- 鉴权：是

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 终端主键 |

成功响应：

```json
{
  "code": 200,
  "message": "OK"
}
```

说明：

- 该接口为逻辑删除，不会物理删除数据库记录

## 7. 患者档案接口

基础路径：`/api/patient`

### 7.1 新建患者档案

- 方法：`POST`
- 路径：`/api/patient`
- 鉴权：是

请求体：

```json
{
  "rfidUuid": "ABCDEFGHJKMN",
  "idCardNo": "110101199003078888",
  "insuranceNo": "YB888888",
  "phone": "13800138000",
  "name": "张三",
  "gender": 1,
  "age": 30,
  "medicalHistory": "无",
  "emergencyContactName": "李四",
  "emergencyContactPhone": "13900139000"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| rfidUuid | string | 否 | 12 位 Base32 大写字符 |
| idCardNo | string | 是 | 18 位身份证号 |
| insuranceNo | string | 否 | 医保卡号，最大 64 位 |
| phone | string | 否 | 大陆手机号 |
| name | string | 是 | 姓名，最大 64 位 |
| gender | number | 否 | `1` 男，`2` 女 |
| age | number | 否 | 0 到 200 |
| medicalHistory | string | 否 | 长期病史信息 |
| emergencyContactName | string | 否 | 紧急联系人姓名 |
| emergencyContactPhone | string | 否 | 紧急联系人电话 |

成功响应：

```json
{
  "code": 200,
  "message": "OK",
  "data": 1926309999999999999
}
```

可能返回：

- `4301`：身份证号已存在
- `4302`：RFID 已绑定其他患者
- `400`：参数校验失败

### 7.2 更新患者档案

### 7.6 按 RFID 拉取患者档案

- 方法：`POST`
- 路径：`/api/patient/fetch-by-rfid`
- 鉴权：是

说明：

- 这是项目核心接口
- 需要同时满足：
  - JWT 合法
  - RFID 校验通过

请求体：

```json
{
  "rfid": "ABCDEFGHJKMNX"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| rfid | string | 是 | 13 位 RFID，前 12 位 UUID + 1 位校验位 |

成功响应：

```json
{
  "code": 200,
  "message": "OK",
  "data": {
    "id": 1926309999999999999,
    "rfidUuid": "ABCDEFGHJKMN",
    "idCardNo": "110101199003078888",
    "insuranceNo": "YB888888",
    "phone": "13800138000",
    "name": "张三",
    "gender": 1,
    "age": 30,
    "medicalHistory": "无",
    "emergencyContactName": "李四",
    "emergencyContactPhone": "13900139000"
  }
}
```

可能返回：

- `4201`：RFID 格式非法
- `4202`：RFID 校验位不匹配
- `4203`：RFID 未绑定任何患者

## 8. 鉴权与联调建议

推荐联调顺序：

1. 调用 `/api/auth/register` 注册终端
2. 调用 `/api/terminal/{id}/status?status=1` 启用终端
3. 调用 `/api/auth/login` 获取 JWT
4. 在后续请求头里带上 `Authorization: Bearer <token>`
5. 调用患者档案相关接口

## 9. curl 示例

### 9.1 注册终端

```bash
curl -X POST 'http://localhost:8080/api/auth/register' \
  -H 'Content-Type: application/json' \
  -d '{

    "terminalName": "住院部3楼-RFID读卡器",
    "secretKey": "secret123"
  }'
```

```bash
curl -X POST 'http://localhost:8080/api/auth/login' \

```

```bash
curl -X POST 'http://localhost:8080/api/patient/fetch-by-rfid' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
```
